# Python Concurrency Homework

## О проекте

Проект представляет собой многопоточный загрузчик логов в memcached (MemcLoad), который парсит и загружает поминутные выгрузки логов трекера установленных приложений. Ключом является тип и идентификатор устройства через двоеточие, значением - сериализованное сообщение с информацией о приложениях и координатах.

## Функциональность

- Парсинг TSV-файлов с логами в gzip-архивах
- Поддержка различных типов устройств: idfa, gaid, adid, dvid
- Многопоточная обработка файлов для повышения производительности
- Обработка файлов в хронологическом порядке
- Сухой прогон без реальной записи в memcached
- Автоматическое переименование обработанных файлов

## Архитектура

- `app/memc_load.py` - основной скрипт загрузки
- `app/appsinstalled_pb2.py` - сгенерированный protobuf-файл с определением структуры данных
- `test/test_memc_load.py` - тесты для основного функционала

## Запуск

### Основной запуск

```bash
python app/memc_load.py --pattern=/path/to/*.tsv.gz
```

### Сухой прогон (без записи в memcached)

```bash
python app/memc_load.py --pattern=/path/to/*.tsv.gz --dry
```

### Запуск с указанием количества потоков

```bash
python app/memc_load.py --pattern=/path/to/*.tsv.gz --workers=10
```

### Запуск тестов

```bash
python -m unittest test/test_memc_load.py
```

## Опции командной строки

- `--pattern` - паттерн для поиска файлов (обязательно)
- `--dry` - сухой прогон без реальной записи в memcached
- `--workers` - количество рабочих потоков (по умолчанию: число ядер * 2)
- `--test` - режим тестирования
- `--idfa`, `--gaid`, `--adid`, `--dvid` - адреса memcached серверов для соответствующих типов устройств
